This page is for field teams preparing to run a live AlphaRelay mission. Use it before deployment to confirm your device, network, roles, and sync plan are ready.

<Info>
  Some current product screens may refer to these modes as Pilot only and Full relay. In this guide, Pilot only is described as a field-device mission, and Full relay is described as a relay mission.
</Info>

## Pre-deployment checklist

- Android field device is charged.
- Controller and drone are connected and tested, if applicable.
- AlphaRelay Android app is installed and updated.
- Operator is signed in.
- License is activated.
- Correct organization is selected.
- Mission Overwatch has been opened once while online on the command device.
- Field Wi-Fi is tested if using LAN relay.
- Internet plan is known if using WebRTC remote live or cloud sync.
- Storage and SD card are checked.
- Roles are assigned: field operator, Overwatch reviewer, and supervisor.
- First training mission is completed before operational use.

## Choose the mission path

| Field condition | Use this path |
|-----------------|---------------|
| One Android field device will run the mission without a same-LAN command laptop | Field-device mission |
| Command laptop is on the same field Wi-Fi as the Android field device | Relay mission with LAN relay |
| Off-site reviewer needs live viewing | WebRTC remote live |
| Field site has no internet | Local mission work, then sync later |
| Agency needs an evidence package | Upload, review, seal, approve, then export |
