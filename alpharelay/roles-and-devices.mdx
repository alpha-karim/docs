AlphaRelay connects field devices, command review, cloud sync, and evidence packaging. This page explains the moving parts before you run a mission.

## Pilot operator

The pilot operator uses an Android tablet or phone with the AlphaRelay pilot app.

The pilot tablet or phone is responsible for:

- Connecting to the drone controller
- **Starting missions** via **Choose Mission Mode** (**Pilot only** or **Full relay**) and kicking off the LAN relay (**full relay**) or remote-live publishing (**pilot only** when reviewers watch off LAN)
- Showing the live pilot interface
- Sending live video and telemetry to Mission Overwatch (LAN relay/WebSocket path **or** configured WebRTC ingest from the pilot)
- Logging quick events from the field
- Logging voice relay events when enabled
- Running optional local AI detection when configured
- Recording or forwarding mission footage
- Queuing pilot-handoff footage locally when cloud upload is not available

## Mission Overwatch operator

The Mission Overwatch operator uses the AlphaRelay web app in a browser.

Mission Overwatch is responsible for:

- **Closing missions from the browser when required** (**Stop Mission** on the pilot is the typical paired shutdown step)
- Consuming **full relay** feeds after dashboard hand-off (LAN metadata prefill/autoconnect, with Connect/scan/manual fallbacks only when needed), or tuning into **Live stream (WebRTC)** when the pilot publishes it
- Logging manual and quick events
- Uploading mission photos
- Reviewing mission footage
- Running **Play with Events**
- Generating reports and offline packages

The Mission Overwatch device can be any supported browser workstation. LAN relay watchers must share the pilot Wi‑Fi/VLAN; reviewers using **Live stream (WebRTC)** intentionally do **not** need direct relay reachability.

## Reviewer or officer

The reviewer or officer works after the mission is closed.

The reviewer is responsible for:

- Confirming the event timeline
- Reviewing uploaded footage
- Tagging post-mission events when needed
- Reviewing and editing the after-action report
- Addressing items requiring officer review
- Sealing the evidence record when ready

## Supervisor

The supervisor reviews the completed mission record when it is ready for approval.

The supervisor may review:

- Mission summary
- Event timeline
- Footage and screenshots
- After-action report
- Chain-of-custody record
- Offline package, when exported for handoff or archive

## Organization admin

The organization admin manages account and setup tasks.

The admin may handle:

- Operator accounts
- Organization settings
- Member names, ranks, roles, and statuses
- Organization time zone
- Organization storage review
- Pilot tablet/phone license keys
- Scenario templates
- Access to mission records

## Core devices

| Device | Used for |
|--------|----------|
| Android pilot tablet or phone | Pilot app, relay, live controls, quick events, footage handoff |
| Drone controller | DJI flight control connection |
| Aircraft | Mission flight and recording |
| Mission Overwatch device | Browser-based command view, event logging, review, reports |
| Cloud account | Sign-in, sync, uploaded footage, shared mission records |
| SD card or aircraft storage | Source footage when manual upload is needed |
