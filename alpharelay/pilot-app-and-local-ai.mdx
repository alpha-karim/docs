## What this is

This is a reference page for the Android field app. It keeps field app details in one place so beginner pages can stay short.

## When to use it

- You need details about the Android field app.
- You are training field operators.
- You need quick event or voice relay details.
- You need optional local AI details.

## Basic steps

1. Open the Android field app.
2. Tap **Choose Mission Mode**.
3. Enter a mission name if needed.
4. Select a scenario template, or choose **No scenario template**.
5. Choose **Field Mission** or **Relay Mission**.
6. Log events during the mission as needed.
7. End the mission with **Stop Mission**.

## What good looks like

- The mission starts from the Android field device.
- Field events appear in the mission timeline.
- Relay Mission connects to Mission Overwatch when the local connection is available.
- Optional local AI events are reviewed by a human.

## Common problems

- Mission Overwatch is expected to start the mission.
- Relay Mission is selected without local network reachability.
- Optional local AI is treated as final evidence without review.

## More detail

The Android field app runs on an approved tablet or phone, connects to the aircraft controller, starts the mission, shows the field interface, logs events, and helps transfer mission footage.

Field Mission means the Android field device is the main tool and live local viewing is not required. Relay Mission means Mission Overwatch connects to the Android field device on the same reachable local network for live viewing and command-side logging.

The field app supports quick event buttons, a quick event wheel, manual event notes, voice relay logging when enabled, and optional local AI events when configured.

When configured, the field app can publish a remote live stream from **Pilot console → Remote live stream → Start remote live**. Remote Mission Overwatch users can watch through **Live stream (WebRTC)** without joining the field device local network.

Local AI is optional on-device detection in the Android field app. When enabled, it samples the same live frames used by the mission feed and can create local AI events with screenshots and metadata. In this build, the pilot setting emits **Person detected** events.

<Warning>
  Local AI supports review and event triage, but it does not replace pilot judgment, agency policy, or required human confirmation.
</Warning>
