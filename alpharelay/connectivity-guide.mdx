This page is for field operators, Mission Overwatch operators, and reviewers planning how a mission will stay connected. Use it before deployment and again when live video, remote viewing, upload, or sync behavior is unclear.

AlphaRelay uses three separate connection paths during a mission:

```text
Android Field Device
  |-- LAN Relay -> Mission Overwatch on same field Wi-Fi
  |-- WebRTC Remote Live -> Off-site reviewers over internet
  `-- Cloud Sync -> Events, footage, reports, evidence records
```

These paths can use the same network, but they do not have to.

## Which path should I use?

| Situation | Recommended path |
|-----------|------------------|
| Same field Wi-Fi with command laptop | LAN Relay |
| Off-site reviewer needs live view | WebRTC Remote Live |
| No internet in field | Local mission, sync later |
| Need full evidence package | Upload footage, review, seal, export |
| Training/demo | First Training Mission |

## What happens if internet drops?

<Warning>
  AlphaRelay is built for imperfect field networks. Local mission work can continue when supported by the current mission mode. Events and footage can queue locally, and cloud sync resumes when connectivity returns. LAN relay, WebRTC remote live, and cloud sync are separate paths, so losing one connection type does not always mean the entire mission stops.
</Warning>

## LAN relay

The pilot tablet or phone and the Mission Overwatch device must be able to reach each other on the same local network.

LAN relay powers:

- Live video in Mission Overwatch
- Telemetry display
- Pilot quick events
- Local event screenshots
- Mission control handoff between the pilot app and browser

If Mission Overwatch cannot connect to the pilot relay:

- Confirm both devices are on the same Wi-Fi network when using the LAN path.
- Confirm the pilot app shows the relay as running for a relay mission.
- Wait for pilot mission sync to populate LAN metadata; then **Tablet/phone IP (LAN)** should fill automatically and connection can proceed **without typing the IP**.
- If the field stays blank or the address looks wrong after sync, leave it empty and click **Connect** to scan the LAN (or paste the IPv4 shown on the pilot screen / a supplied tunnel URL).
- Trust the relay certificate if the browser prompts for it.

If reviewers are deliberately off-LAN and remote live is configured instead, rely on **Live stream (WebRTC)** rather than troubleshooting local discovery.

## WebRTC remote live

WebRTC remote live is the internet path for reviewers who are not on the field Wi-Fi.

Remote live requires:

- Internet access from the Android field device.
- Remote live started from the pilot app when configured.
- Reviewers to open Mission Overwatch and use **Live stream (WebRTC)** when offered.

Use WebRTC remote live when an off-site reviewer needs live situational awareness. Use LAN relay when the command laptop is on the same field Wi-Fi and the team wants the lowest-latency local path.

## Cloud sync

Internet connectivity is needed for cloud-backed work.

Cloud sync powers:

- Operator sign-in
- Mission sync
- Uploaded mission footage
- Uploaded mission photos
- Playback links
- Cloud evidence records
- Organization data
- Cross-device review
- Remote live stream setup when configured

If internet is unavailable, AlphaRelay can still support local mission work, but cloud sync waits until the device is online again.

## What works offline or on local network only

Depending on the current mission state and device setup, AlphaRelay can continue to support:

- Local relay viewing
- Event logging
- Screenshots captured from the pilot relay
- Local mission state
- Deferred pilot-handoff footage queueing
- Recently validated pilot tablet/phone licensing within the app's offline validation window

Keep AlphaRelay open when possible so queued changes can sync when connectivity returns.

## Mission Overwatch offline app shell

Mission Overwatch installs a service worker on first online visit and can also be installed as a PWA from the **Install app** button when the browser supports it. After that, the static Mission Overwatch UI can load again even without internet access on the same browser profile.

The offline app shell only covers the page itself. Cloud sign-in, mission sync, footage uploads, and playback links still need the network and a signed-in session.

**Field SOP:** open Mission Overwatch once while online (or install it) on each laptop, tablet, or phone browser profile that will run a mission, so the service worker has time to cache the UI before the team reaches a low-connectivity site.

## What queued status means

You may see messages such as:

- **Queued for cloud sync**
- **Loading playback link**
- A pending mission video count
- A pilot tablet or phone upload banner

These usually mean AlphaRelay has local mission data or pilot-handoff footage that still needs to reach the cloud.

When you see these states:

1. Stay signed in.
2. Keep the browser or pilot app open.
3. Move the device to a network with internet access when practical.
4. Use **Upload now** on the pilot tablet or phone if it appears and the device is online.
5. Wait for the upload and playback link to finish before final review.

## Switching networks after a mission

Field networks are often optimized for the pilot relay, not internet upload. After the mission:

1. Land and close the mission.
2. Confirm critical events and screenshots are present.
3. Stop the relay when it is safe to do so.
4. Move the device to an internet-connected network.
5. Let cloud sync and footage upload complete.

If pilot-handoff footage is queued locally, AlphaRelay retries upload when internet access returns.
