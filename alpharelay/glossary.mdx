## Alpha AI

AlphaRelay's report generation assistant. It drafts after-action report content from the structured mission record, events, screenshots, footage references, and custody context.

## Chain of custody

The evidence history for a mission. It helps show what media and records were captured, reviewed, sealed, exported, or handed off.

## Cloud sync

The process that saves local mission changes, events, screenshots, footage records, and review state to AlphaRelay cloud services.

## Event

A timestamped mission entry. Events can be logged from the pilot tablet or phone, Mission Overwatch, or post-mission footage review.

## Event screenshot

An image linked to an event. It may come from the live pilot relay or from a captured video frame during post-mission tagging.

## Local AI

Optional on-device detection in the Android pilot app. It samples the live relay frames and can create local AI events with screenshots and metadata. In the current build, the pilot setting is fixed to person-detected events.

## Mission Dashboard

The web app landing area after sign-in. It shows active and closed missions, mission search, operational metrics, report queues, and after-action report actions.

## Mission Console

Mission Console is the technical page name for the Mission Overwatch workspace.

Operators open it from **Open in console** on the dashboard row for that flight. Starting a mission stays on the Android pilot app; Mission Overwatch handles relay/WebRTC feeds when available, structured logging, uploads, playback, Alpha AI reporting, approvals, sealing, and offline export.

## Mission photo

An uploaded image linked to a mission. Mission photos can be timeline items or evidence-only items and can appear in reports, custody records, storage summaries, and offline packages.

## Mission Overwatch

The focused mission workspace in the browser (opened from dashboard **Open in console**). It consumes synced mission metadata so the LAN relay URI can preload for relay missions, exposes **Live stream (WebRTC)** when configured, manages events, uploads, custody, playback, Alpha AI reporting, approvals, sealing, and export. Android always starts the mission; Overwatch participates after dashboard handoff.

## Offline package

A downloadable ZIP archive containing a point-in-time copy of the mission record. It can include the offline viewer, report, custody data, certificate, timeline, footage, and screenshots.

## Pilot app

The AlphaRelay Android app used on the pilot tablet or phone. It connects to the drone controller, starts the relay, displays pilot controls, logs quick events, and helps transfer mission footage.

## Pilot relay

The local connection served by the pilot tablet or phone so Mission Overwatch can view the live feed and mission data on the same local network.

## Remote live

A configured live-stream path that lets Mission Overwatch view the active pilot feed over the internet instead of the pilot device LAN.

## Play with Events

The footage review view that combines video playback, event markers, event screenshots, multi-angle controls, sync tools, and post-mission event tagging.

## Primary footage

The main reference video for the mission timeline. Event markers are mapped against this timeline first.

## Secondary angle

Additional mission footage from another source or angle. Secondary clips can be synchronized to the primary footage so events line up across views.

## Sync offset

The time adjustment used to align a secondary angle with the primary footage timeline.

## Tag event

A post-mission review action in **Play with Events**. It creates a new event at the current video frame and stores a screenshot from that frame.
