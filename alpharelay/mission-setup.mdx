## Network and hardware

Before starting a mission:

- Put the pilot tablet and command device on the same Wi-Fi network.
- Connect the pilot tablet to the drone controller.
- Power on the drone controller.
- Power on the aircraft.

The command device can be a laptop, desktop, another tablet or phone, or the same device running the app.

## Start the relay and mission console

**On the pilot tablet**

1. Choose the mission mode.
2. Use **Full relay** when Mission Overwatch will connect to this tablet for a live browser view.
3. Use **Pilot only** when the pilot app should create a field mission without a live Mission Overwatch browser session.
4. Tap **Start Relay + DJI**.
5. Complete any license step if prompted.

**In the web app**

1. From the home or command dashboard, click **Open Mission Console**.
2. Under **Mission**, enter the mission name.
3. Select a **Scenario Template**, if your team uses one.
4. Click **Start Mission**.
5. Under **Live Overwatch Screen**, leave **Tablet IP (LAN)** blank to scan the LAN, or confirm the saved tablet IP if one is already shown.
6. Click **Connect**. Mission Overwatch scans for the LAN relay or connects to the entered tablet IP.

<Tip>
  Manual IP entry is a fallback. Use it if LAN scan does not find the tablet, or paste a full `ws://` or `wss://` URL if your team is using a tunnel.
</Tip>

If the browser asks you to trust the pilot relay certificate:

1. Click **Trust Relay Certificate**.
2. Complete the trust page in the new tab.
3. Return to Mission Overwatch.
4. Connect again.

## Remote live option

If your team has remote live streaming configured, the pilot app can start a remote live stream for an active mission. Mission Overwatch can then switch the feed source to **Live stream (WebRTC)** instead of using the local relay feed.

Use the local relay when the command device is on the same network as the tablet. Use remote live when remote reviewers need to watch without joining the tablet LAN.
