## What this is

This reference keeps setup details for teams that need more than the beginner pages.

## When to use it

- You need hardware and network reminders.
- You need details about Mission Overwatch connection behavior.
- You need to troubleshoot a Relay Mission.
- You need to understand remote live at a high level.

## Basic steps

1. Prepare the Android field device, controller, and aircraft.
2. Choose Field Mission or Relay Mission from the Android field app.
3. Complete the license step if prompted.
4. Open the active mission from **Mission Dashboard**.
5. Use Mission Overwatch if the selected workflow needs it.

## What good looks like

- The mission starts from the Android field device.
- Relay Mission devices share a reachable local network.
- Mission Overwatch opens the active mission from the dashboard.
- Review tools remain available after mission closeout.

## Common problems

- The team expects browser mission start.
- Relay Mission local connection is attempted across unreachable networks.
- Remote live is confused with Relay Mission local viewing.

## More detail

Before starting a mission, connect the Android field device to the controller, power on the controller, and power on the aircraft when applicable.

Mission records are created on the Android field device. Mission Overwatch does not start missions from the browser.

For Relay Mission, AlphaRelay syncs mission metadata so Mission Overwatch can attempt connection after the active mission is selected. If connection does not succeed, use **Connect** with the field blank to scan, or enter the address shown on the Android field device. If the browser prompts for TLS, use **Trust relay certificate**, complete trust, return, and connect again.

When remote live streaming is configured, the field app publishes from **Pilot console → Remote live stream → Start remote live**. Reviewers then use **Live stream (WebRTC)** in Mission Overwatch when offered.
