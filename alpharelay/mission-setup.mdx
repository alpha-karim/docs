## Network and hardware

This page is for field teams preparing to start a live AlphaRelay mission. Use it before launch to confirm hardware, mission mode, Mission Overwatch access, and the live-view path.

Before starting a mission:

- Put the pilot tablet or phone and command device on the same Wi‑Fi network **when using the LAN relay** (unless the team relies on configured **remote live** instead).
- Connect the pilot tablet or phone to the drone controller.
- Power on the drone controller.
- Power on the aircraft.

The pilot app runs on an approved Android tablet or phone. The command device can be a laptop, desktop, tablet, phone, or another browser profile.

## Start the mission from the Android pilot tablet or phone

Mission records are created on the pilot device. **Mission Overwatch no longer starts missions from the browser** — always begin from the app.

On the pilot tablet or phone:

1. Tap **Choose Mission Mode**.
2. In the **Choose mission mode** overlay, optionally type a mission name (otherwise defaults apply), pick a scenario template (or **No scenario template**), then tap one of:
   - **Field-device mission** — the mission can run from the Android field device without requiring a LAN Mission Overwatch session. Remote reviewers can still use **Live stream (WebRTC)** when that path is configured and running on the pilot.
   - **Relay mission** — the Android field device connects to Mission Overwatch on the same field Wi-Fi for live LAN viewing and command-side event logging.
3. Complete the license step if prompted.

The app syncs mission metadata (including the tablet’s LAN IPv4 and relay URLs when available) so the browser can reconnect without you retyping addresses.

<Tip>
  Long-press **Choose Mission Mode** to open the **AlphaRelay license** settings without starting the relay.
</Tip>

## Open Mission Overwatch from the dashboard

After the pilot starts a mission and cloud sync catches up, the mission appears under **Active missions** on the **Mission Dashboard**.

1. Select the active mission. Mission Overwatch opens for that mission.

### LAN relay

For relay missions, use the **Live Overwatch Screen** card:

- AlphaRelay fills **Tablet/phone IP (LAN)** from the synced mission metadata and attempts relay connection automatically when the pilot address is known.
- If connection does not succeed, use **Connect** with the field blank to **scan the LAN**, or enter the IPv4 shown on the pilot UI as a fallback. Optional **Relay token** remains for deployments that enforce it.
- If the browser prompts for TLS, use **Trust relay certificate**, complete trust, return here, then connect again.

Overwatch uses `wss://` and port `8789` when you supply only an IPv4. For tunnels or unusual routing, paste a full `ws://` or `wss://` URL.

### Field-device mission plus remote reviewers

When the mission was started as **Field-device mission**, Mission Overwatch hides the LAN relay/event console blocks that assume a bridged relay, but **Playback / events / footage / reports** remain for review.

If remote live streaming is configured, the pilot publishes from **Pilot console → Remote live stream → Start remote live**; reviewers then use **Live stream (WebRTC)** in the feed shell when offered.

Use the LAN relay path when the command device is on the same network as the pilot tablet or phone (lowest latency). Use **Live stream (WebRTC)** when reviewers are off the pilot LAN.
